# 会议纪要

Use this reference when the user asks for 会议纪要, 会议记录整理, meeting minutes, discussion summary, or converting meeting notes into a formal internal record.

## Goal

Convert meeting notes into a clear record of topics, decisions, assigned actions, and follow-up responsibilities.

## Recommended Structure

标题：会议纪要

会议时间：`【会议时间】`

会议地点：`【会议地点】`

主持人：`【主持人】`

参会人员：`【参会人员】`

记录人：`【记录人】`

会议主题：`【会议主题】`

一、会议基本情况

二、会议研究事项

三、会议议定事项

四、下一步工作安排

## Content Rules

- Put factual meeting context in "会议基本情况".
- Put discussion topics and main views in "会议研究事项".
- Put final decisions in "会议议定事项".
- Put owners, deadlines, and deliverables in "下一步工作安排".
- Do not convert a discussion opinion into a decision unless the source clearly says it was agreed or decided.

## Action Item Pattern

Use this pattern for follow-ups:

`由【责任部门/责任人】负责，于【完成时限】前完成【具体事项】，并形成【成果形式】。`

If facts are missing, keep placeholders.

## Writing Style

- Use formal but direct wording.
- Prefer "会议指出 / 会议认为 / 会议要求 / 会议明确" only when it fits the source.
- Avoid overusing official phrases when the source is a practical internal meeting.
- Keep decisions and action items easy to scan.

## Minimal Text Template

一、会议基本情况
说明会议召开的背景、目的和参会范围。

二、会议研究事项
围绕每个议题概括讨论情况。

三、会议议定事项
列明已经明确的决定、原则、口径或方案。

四、下一步工作安排
逐项列明责任人、完成时限和交付成果。
